# GLx Feature Closure Matrix

## Status

Accepted current-state matrix, 2026-05-09.

This document is the checked-in feature-closure ledger for GLx. It describes the current transitional renderer against the final contract in [GLX_FINAL_CONTRACT.md](GLX_FINAL_CONTRACT.md). It is not a release claim: rows marked `covered` still need to remain covered by the blocking runtime proof before GLx can replace the legacy OpenGL renderers.

## Status Legend

| Status | Meaning |
|---|---|
| covered | GLx has an implemented path plus current diagnostics, logic tests, or sweep coverage that can keep the feature visible during RC review. |
| partially covered | GLx can exercise some or all of the feature through a shared compatibility path, a gated path, diagnostics-only proof, or a subset of tiers, but promotion still needs ownership or runtime evidence. |
| missing | GLx does not yet have the required GLx-owned implementation or proof path. |

Rows must use exactly one of `covered`, `partially covered`, or `missing`. Empty, unknown, TBD, or mixed status rows are not allowed.

## Promotion Interpretation

The matrix is intentionally conservative. A row can be `covered` for current tracking while GLx still remains promotion-gated; default promotion also requires the [Promotion Rules](GLX_FINAL_CONTRACT.md#promotion-rules), blocking runtime artifacts, and the ownership-proof profile. A row marked `missing` is not ambiguous. It is a named blocker that must either be implemented, deliberately removed from GLx replacement scope with maintainer approval, or kept behind a retained legacy renderer path.
